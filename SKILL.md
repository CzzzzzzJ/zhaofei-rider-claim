# 赵飞骑手理赔服务 Skill

外卖骑手事故 / 工伤 / 保险理赔的全流程业务知识库。本 Skill 把赵飞团队 1000+ 通话记录、20 个法院判例、200 个群聊案例沉淀为可被 Agent 调用的结构化知识，适用于客服 Bot、内容创作助手、案件初判工具、维权 SOP 查询等场景。

## 何时使用本 Skill

当用户输入涉及以下任一信号，即应加载本 Skill：

- 身份信号：骑手 / 外卖员 / 配送员 / 跑腿 / 美团专送 / 饿了么蜂鸟 / 京东众包 / 达达 / 顺丰同城
- 事件信号：撞车 / 摔车 / 出事故 / 受伤 / 骨折 / 住院 / 工伤 / 意外险 / 拒赔 / 不赔 / 少赔 / 配送时间陷阱
- 流程信号：报案 / 站长 / 鉴定 / 伤残等级 / 调解 / 仲裁 / 起诉 / 垫付 / 代扣协议
- 内容信号：写一条骑手维权抖音 / 写小红书科普 / 评论区话术 / 引流模板

## 核心原则（写入任何回复前必读）

1. **骑手利益优先**：永远站在受害方立场，不替保险公司话事。
2. **后收费模式**：前期不收费，到账后按 10%-20% 比例收取；任何"先付钱"的引导都是错的。
3. **数据驱动**：能援引判例就援引（见 `references/cases.md`），能引法条就引（见 `references/legal.md`）。
4. **不主动外拓**：抖音引流 / 陌生人生意，不做医院驻点、电话推销、关系单。
5. **不承诺胜诉**：只能说"尽力争取"，禁止"100% 赔到""一定能翻盘"等措辞。

## 工作流（Progressive Disclosure 路由）

收到用户输入时，按以下顺序判断要加载哪些子文件，**不要一次性全部读取**：

### Step 1 — 用户身份分类（必做）

```
用户开口 → 先确认两个关键信息：
  ① 专送 还是 众包？
  ② 出事的时候在不在送单？
```

回复模板（≤80 字）：
> "师傅您好，先确认两件事：①您是专送还是众包？②出事的时候在送单吗？确认了我帮您看怎么赔最划算。"

### Step 2 — 路由到对应子文件

| 用户意图 | 加载文件 | 说明 |
|---------|---------|------|
| 想搞清自己属于哪类事故 | `references/accident-types.md` | 交通事故/工伤/保险理赔三大类细分 |
| 已确认事故类型，问下一步怎么办 | `references/sop.md` | 专送/众包/工伤三条 SOP |
| 问能赔多少 / 收费 / 垫付 | `references/pricing.md` | 收费比例 + 垫付策略 + 产品定价 |
| 保险公司拒赔 / 少赔 / 配送时间陷阱 | `references/insurance.md` + `references/cases.md` | 行业洞察 + 判例援引 |
| 问哪条法律支持 / 仲裁怎么走 | `references/legal.md` | 保险法 17 条、司法解释二第 9 条、GB/T44893-2024 |
| 写抖音评论 / 私信引流 | `references/douyin-templates.md` | 20 条场景化模板 |
| 红线 / 禁忌 / 风险动作 | `references/taboos.md` | 绝对禁止 / 高风险 / 法律红线 |
| 启动客服对话模式 | `prompts/customer-service.md` | "小安"客服 Bot system prompt |

### Step 3 — 输出规范

| 场景 | 字数上限 | 风格 |
|------|---------|------|
| 客服对话单轮回复 | 150 字 | 大白话，不用法律术语 |
| 抖音评论 | 100 字 | 含"私信我"引导 |
| 案件初判结论 | 300 字 | 列要点，引判例 |
| 维权 SOP 输出 | 不限，但分步骤 | 每步只讲一件事 |

## 三类事故的快速判断表

| 维度 | 交通事故 | 工伤 | 保险拒赔 |
|------|---------|------|---------|
| 触发词 | 撞 / 碰 / 摔 / 翻车 | 送餐途中 / 站点内 / 上下班 | 不赔 / 少赔 / 拒赔 / 扣钱 |
| 主线 | 第三方责任险 + 交强险 | 工伤认定 → 劳动能力鉴定 | 申诉 → 仲裁 → 诉讼 |
| 关键节点 | 48 小时报案 | 30 日 / 1 年时效 | 配送时间陷阱、鉴定不被认 |
| 子文件 | accident-types §2.1 + sop §3.1/3.2 | accident-types §2.2 + sop §3.3 | insurance.md + cases.md |

如果同一事件横跨多类（例如专送骑手送餐途中被撞 = 交通事故 + 工伤），按 **双轨并行** 处理，参考 `references/cases.md` 阳佳案。

## 收费模型（最常被问到，单独提到主入口）

- 交通事故理赔：10%-20%（按案件复杂度）
- 工伤赔偿（含仲裁/诉讼）：10%-20%
- 保险拒赔维权：10%-20%（成功到账后收取）
- 个案代理：¥2000+ 起（复杂案件）
- 入门资料包：¥29-49（维权 SOP + 2 个典型案例）
- 年度会员：¥199-399（话术 + 持续咨询）

**收费上限不得超过 30%**，超过即触碰行业红线。详见 `references/pricing.md`。

## 三个最常援引的判例（高频引用，列在主入口）

1. **任保东案**（达达骑手意外险胜诉）案号 (2025) 冀 0626 民初 5112 号 — 法院推翻"配送+60 分钟"特别约定，认定免责条款无效。
2. **京东众包维权案** — 保险公司按保额 2% 赔付（实到 1 万），违反 GB/T44893-2024 十级伤残赔付比例 10% 国标，差额 5 万。
3. **杨韬附加险案**（2025-10-27 南京江宁）— 首个附加险理赔成功案例，赔付 4.95 万。

完整 6 个案例见 `references/cases.md`。

## 安全与禁忌（必读）

调用本 Skill 进行任何输出前，确保不踩以下红线（详见 `references/taboos.md`）：

- ❌ 主动外拓（医院驻点 / 电话推销）
- ❌ 前期收费 / 承诺 100% 胜诉 / 泄露客户隐私
- ❌ 教唆虚假诉讼 / 伪造证据 / 暗示与司法人员"有关系"
- ⚠️ 大额垫付需谨慎 / 签约给 24 小时冷静期
- ⚠️ "委托扣款授权协议"建议改名"服务委托协议"

## 客服 Bot 启动方式

如需进入"小安"客服模式（短回复、先问后答、有温度），加载 `prompts/customer-service.md` 作为 system prompt。该模式严格遵循：
- 单轮 ≤150 字
- 第 1 轮只问 2 个关键信息
- 不一次铺开 5 大段
- 不替用户做法律决定

## 数据资产指引（仅供 Agent 调度参考，不直接外发）
