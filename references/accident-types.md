# 事故类型判断

骑手事故的三大类与子类。客服 Bot / 案件初判工具用本表先做归类，再决定走哪条 SOP。

## 1. 交通事故类

| 子类 | 特征 | 典型走向 |
|------|------|---------|
| 机动车事故 | 与汽车/货车碰撞 | 第三方交强险 + 商业险 |
| 非机动车事故 | 与电动车/自行车碰撞 | 对方非机动车赔偿 + 自身意外险 |
| 单方事故 | 自己摔倒 / 撞墙 / 撞物 | 仅能走自身意外险 / 工伤 |
| 骑手有责事故 | 闯红灯 / 逆行 / 超速 | 责任划分影响赔付比例 |

**判断要点**：
- 是否有交警到场出具《道路交通事故认定书》— 决定责任归属
- 对方是否有保险、保额多少 — 决定赔付天花板
- 单方事故 → 没有第三方责任险可走，必须走工伤 / 意外险

## 2. 工伤事故类

| 子类 | 特征 | 时效 |
|------|------|------|
| 送餐途中受伤 | 工作时间 + 工作场所 + 工作原因（"三工"齐备） | 单位 30 日内申请，个人 1 年内申请 |
| 站点内受伤 | 在站点装货/卸货 / 早会 | 同上 |
| 上下班途中 | 合理路线 + 合理时间，非本人主要责任 | 同上 |

**判断要点**：
- 是否有劳动关系 / 劳务关系证明（合同、工资流水、考勤、骑手 App 截图）
- 是否在"配送时间"内 — 直接影响保险公司认不认
- 单位不申请 → 骑手 1 年内可自行申请，逾期作废
- 未缴社保但有雇主责任险 → 仍可追偿

## 3. 保险理赔类

| 险种 | 适用平台 | 关键陷阱 |
|------|---------|---------|
| 骑手意外险 | 美团 / 饿了么 | "配送时间"特别约定 — 送完单 60 分钟内才算 |
| 雇主责任险 | 专送站点 | 投保金额是否真实、是否合规 |
| 附加险 | 部分站点 | 赔付比例问题，业务员常以"按保额 2%"敷衍 |
| 新职伤 | 试点城市（北京、上海、江苏等） | 流程区别于传统工伤，需确认本地试点状态 |

**判断要点**：
- 拒赔 ≠ 没戏：先拿拒赔通知书，再根据理由判断翻盘空间
- "鉴定不予认可" — 最常见的拖延理由，可申请重新鉴定或诉讼
- "管辖权异议" — 保险公司常用拖延手段，骑手可坚持事故地或居住地管辖

## 跨类事件处理

骑手送餐途中被汽车撞伤 → 同时是 **交通事故 + 工伤**，建议双轨并行（参考 `cases.md` 阳佳案，工伤 2.8 万 + 交通事故 3.6 万）。

| 双轨可叠加项 | 双轨不可叠加项 |
|------------|-------------|
| 一次性伤残补助金（工伤） | 医疗费（实报实销，不重复赔） |
| 第三方人身损害赔偿（交通事故） | 误工费（择一就高） |
| 精神抚慰金（交通事故） | — |
